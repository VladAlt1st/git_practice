1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout file(.) или git restore file(.)

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged file

3.Как отменить ошибочный commit?
git reset HEAD~1 - можно откатится на один коммит и продолжить оттуда,
таким образом отменяя ошибочный коммит, если еще не было пуша.
Если пуш уже был, то лучше сделать git revert HEAD что бы откатить ошибочный коммит,
после чего сделать нужный коммит и запушить

4.Как создать новую ветку и переключится в нее?
git branch name и git checkout name или git checkout -b name

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
Если при разработке основная ветка не менялась и мы вливаем в нее ветку которая ушла вперед, то произойдет fast-forward merge, простое перемещенее указателя на более новый вариант.
Если же две ветки разрабатываються параллельно и при слиянии происходит конфликт, то создается новый коммит который объединяет изменения, то есть происходит non fast-forfard merge
 
6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
Если один и тот же файл был модифицирован в разных ветках. Нужно зайти в редактор, выбрать нужные изменения, добавить файл в индекс и сделать коммит

7.Можно ли удалить ветку? Как? 
git branch -d name (нужно быть на другой ветке)

В ответах на вопросы 1-4, 7 необходимо указать конкретные команда git. 
На 5 и 6 вопросы дать объяснение своими словами.
